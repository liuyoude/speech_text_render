## 2. Emphasis 权重是否需要调整

### 当前配置

```
w_energy=0.4, w_duration=0.3, w_pitch=0.3, threshold=1.0
```

### 统计

- 417 词中检测到 17 个 emphasis = **4.1% 的词**
- composite_z 范围：[1.00, 2.22]，均值 1.55，标准差 0.42
- 分布集中在 1.0-2.2，无极端值

### 结论

**当前权重和阈值无需调整**。4% 的 emphasis 密度合理（过密会成为噪声，过稀会丧失信息）。阈值 1.0 恰好过滤了大部分普通词，保留了真正突出的词。

可在 Stage 3 大数据集校准后复核：如果大数据集上 emphasis 密度 > 8% 可考虑提高 threshold 到 1.2；如果 < 2% 可降低到 0.8。

## 3. Z-score 连续值范围是否集中在 [-3, 3]

### 统计

| 控制符 | n | 均值 | 标准差 | 范围 | 撞到 +-3 截断的比例 |
|--------|---|------|--------|------|------------------|
| speed | 35 | +0.53 | 2.00 | [-3.0, +3.0] | **31%** |
| volume | 34 | -0.92 | 1.30 | [-3.0, +2.1] | 9% |
| pitch | 8 | +0.81 | 0.72 | [-0.72, +1.89] | 0% |
| emphasis | 17 | +1.55 | 0.42 | [+1.0, +2.22] | 0% |
| break | 42 | -- | -- | [0.2, 1.2]s | N/A (绝对秒数) |

### 分析

- **speed 31% 撞截断** -- 这是最突出的问题。原因：当前 population stats 样本极小（zh_read n=13, en_read n=9），std 偏窄（zh_read speed std=0.45），导致稍快/稍慢的音频 z-score 就超出 +-3。
- **volume 9% 撞截断** -- 中等程度，部分 whisper 类音频能量极低导致。
- **pitch 和 emphasis** -- 范围健康，未触及截断。
- **break** -- 使用绝对秒数而非 z-score，范围 [0.2, 1.2]，对 tokenizer 友好。

### 结论

**Z-score 截断到 [-3, 3] 的设计是正确的**，确保了 tokenizer 友好的数值范围。speed 的高截断率是 population stats 样本量不足（n=5~13）造成的，**Stage 3 用 AISHELL-3/ESD/LJSpeech 大数据集校准后 std 会更合理，截断率会自然下降**。当前无需修改截断范围或提取逻辑。
